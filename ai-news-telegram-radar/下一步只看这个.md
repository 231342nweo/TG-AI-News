# 下一步只看这个

我已经把项目收成几个可以直接双击的入口。你先不用管代码。

## 你现在要做的事

1. 打开这个文件夹：

   `/Users/zhuzhiqin/Documents/小号AI信息源/ai-news-telegram-radar`

2. 先双击：

   `一键运行Demo.command`

   这一步不需要网络、不需要 Telegram，只是确认流程能跑。

3. 再双击：

   `一键抓取真实新闻.command`

   这一步会抓真实 RSS 信源，并生成一版 AI 新闻摘要预览。

   频道消息会使用中文字段、中文看点，并统一显示北京时间；原文标题保留，方便核对来源。

4. 去 Telegram 做两件事：

   - 找 `@BotFather` 创建 bot，拿到 bot token
   - 把这个 bot 加到你的频道，并设成管理员

5. 双击：

   `一键配置并测试Telegram.command`

   它会让你输入：

   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`

   公开频道的 `TELEGRAM_CHAT_ID` 通常就是 `@频道用户名`。

6. 测试消息收到后，双击：

   `一键发送AI新闻到Telegram.command`

7. 如果要把英文标题/摘要自动翻成中文，双击：

   `一键配置OpenAI翻译.command`

   它会让你输入 API Key、Base URL 和模型名。

   如果你用 LiteLLM Virtual Key：

   - API Key 填 LiteLLM Virtual Key
   - Base URL 填 LiteLLM Proxy 地址，通常以 `/v1` 结尾
   - 模型名填这个 virtual key 允许调用的模型名或 alias

   没有这个 key 时，系统会继续使用“原文标题 + 中文看点”，不会影响频道正常发送。

## 我需要你验证的地方

- Telegram 频道是否收到测试消息
- Telegram 频道是否收到第一条 AI 新闻摘要
- 摘要是否太吵、太学术、太英文、太长
- 开启 OpenAI 翻译后，中文标题和中文摘要是否自然

如果你把 BotFather token 和频道用户名发给我，我也可以直接帮你把 `.env` 配好并测试；但 token 属于私密信息，更推荐你在本机弹出的配置窗口里输入。

如果 token 已经发到聊天里，建议确认频道跑通后，在 `@BotFather` 里重新生成一次 token，再双击 `一键配置并测试Telegram.command`，只在本机窗口里输入新 token。
